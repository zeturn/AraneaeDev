# Contributing

Thank you for helping improve this project. This guide keeps changes reviewable and easy to maintain.

## Before You Start

- Open an issue for larger behavior changes or architecture changes.
- Keep pull requests focused on one problem at a time.
- Check existing issues and pull requests to avoid duplicate work.

## Development Workflow

1. Create a branch from the default branch.
2. Install the dependencies for the component you are changing.
3. Make the smallest change that solves the problem.
4. Run the relevant tests, builds, linters, or manual checks.
5. Update documentation when setup, configuration, behavior, or APIs change.

## Pull Request Checklist

- Describe what changed and why.
- Include screenshots or API examples for user-visible changes when useful.
- Note the verification you ran, including any checks that could not be run.
- Avoid committing secrets, generated build outputs, dependency caches, or local environment files.

## Reporting Bugs

Please include:

- A clear description of the problem.
- Steps to reproduce.
- Expected and actual behavior.
- Environment details, logs, or screenshots when relevant.

## Security Issues

Do not report vulnerabilities in public issues. Follow `SECURITY.md` instead.
